# Change Record: CHG-001

## Title: Implement Connection Pool Limits for Gateway

**Status:** Completed  
**Date:** 2026-01-16  
**Type:** Hotfix  
**Related Incident:** INC-001

## Summary

Implemented connection pool limits to prevent memory exhaustion incidents like INC-001.

## Changes

| Setting | Before | After |
|---------|--------|-------|
| Max Connections | Unlimited | 100 |
| Connection Timeout | None | 30s |
| Memory Limit | None | 2GB |
| Idle Timeout | None | 60s |

## Configuration

```yaml
gateway:
  connection_pool:
    max_connections: 100
    timeout_seconds: 30
    idle_timeout_seconds: 60
  resources:
    memory_limit: 2GB
```

## Validation

- Load tested at 200% expected peak traffic
- Memory stayed below 1.5GB
- No connection leaks observed over 24h soak test

## References
- Resolves: INC-001
- Related ADR: ADR-001 (Gateway naming)
